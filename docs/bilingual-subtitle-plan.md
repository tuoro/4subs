# 双语字幕新架构说明

## 当前可跑链路

目前项目已经可以跑通以下流程：

1. 扫描本地媒体目录
2. 创建任务并进入后台执行器
3. 先查找同名外挂字幕或内嵌文本字幕轨
4. 找不到字幕则提取音频并调用远程 ASR 转写
5. 转换为标准 `SRT`
6. 调用 `DeepSeek API` 逐批翻译
7. 生成双语 `SRT` 与 `ASS` 到输出目录
8. 在任务详情页人工预览和校对结果字幕

## 当前架构

### `internal/jobrunner`

负责后台任务串行执行，包括阶段推进、状态更新和失败落库。

### `internal/media`

负责三件事：

- 从视频附近查找同名外挂字幕
- 通过 `ffmpeg` 提取内嵌文本字幕轨
- 在需要时提取音频给 ASR 使用

### `internal/subtitle`

负责：

- 解析 `SRT`
- 按时间轴保留字幕块
- 合并原文与译文生成双语 `SRT`
- 生成适合播放器样式化显示的双语 `ASS`

### `internal/translator/deepseek`

当前使用 `DeepSeek` 官方兼容的 `chat/completions` 接口，并约束返回 JSON 结构，避免翻译时丢失字幕顺序。

### `internal/asr/openai`

当前使用 OpenAI 兼容的音频转写接口，按 `verbose_json + segment` 方式获取带时间轴的转写结果。

### `任务详情页`

前端提供源字幕只读预览、双语 `SRT/ASS` 可编辑预览，以及在线保存能力，适合人工校对最后一轮成品。

## 当前限制

这一版更适合：

- 已经带外挂字幕的视频
- 容器内嵌文本字幕轨的视频
- 没有字幕但可以通过远程 ASR 转写的视频

暂未覆盖：

- 图片字幕 OCR
- 多人协作审校
- 任务取消与并发调度

## 后续扩展建议

- 增加任务取消与并发控制

### 第二优先级

- 增加任务日志页面
- 增加批量处理模板
- 增加字幕版本管理

### 第三优先级

- 增加多翻译供应商切换
- 增加协作审校流程
- 增加审核历史记录
