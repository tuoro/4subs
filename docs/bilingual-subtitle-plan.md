# 双语字幕新架构说明

## 项目定位

`4subs` 现在不再是旧的字幕下载聚合器，而是一个面向本地媒体的双语字幕生成平台。

标准流程：

1. 扫描本地媒体目录
2. 创建字幕任务
3. 提取音频到工作目录
4. 调用语音识别生成原文字幕
5. 调用 `DeepSeek API` 逐条翻译
6. 生成双语 `SRT/ASS`
7. 提供人工预览和修正

## 当前已落地骨架

### 后端

- `Go + Chi`
- `SQLite` 存储设置、媒体清单、字幕任务
- 新 API 已围绕“扫描、设置、任务、流水线”组织

### 前端

- `Vue 3 + PrimeVue`
- 工作台分为：总览、流水线、设置 三个页面

### 容器

- Docker 镜像已内置 `ffmpeg`
- 支持挂载本地媒体目录
- 支持 GitHub Actions 构建 GHCR 镜像

## 核心模块规划

### `internal/library`

负责本地媒体发现与基础元数据登记。

### `internal/pipeline`

负责描述任务处理阶段，后续可以扩展成真正的执行编排器。

### `internal/translator`

负责统一翻译层接口，当前已经预留 `DeepSeek` 客户端骨架。

### `internal/server`

负责对外暴露控制面 API，并托管前端页面。
