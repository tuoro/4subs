# 双语字幕新架构说明

## 当前可跑链路

目前项目已经可以跑通以下流程：

1. 扫描本地媒体目录
2. 创建任务并进入后台执行器
3. 提取外挂字幕或视频内嵌文本字幕
4. 转换为标准 `SRT`
5. 调用 `DeepSeek API` 逐批翻译
6. 生成双语 `SRT` 到输出目录

## 当前架构

### `internal/jobrunner`

负责后台任务串行执行，包括阶段推进、状态更新和失败落库。

### `internal/media`

负责两件事：

- 从视频附近查找同名外挂字幕
- 通过 `ffmpeg` 提取内嵌文本字幕轨

### `internal/subtitle`

负责：

- 解析 `SRT`
- 按时间轴保留字幕块
- 合并原文与译文生成双语 `SRT`

### `internal/translator/deepseek`

当前使用 `DeepSeek` 官方兼容的 `chat/completions` 接口，并约束返回 JSON 结构，避免翻译时丢失字幕顺序。

## 当前限制

这一版没有接 `ASR`，所以它更适合这两类视频：

- 已经带外挂字幕的视频
- 容器内嵌文本字幕轨的视频

如果后续要覆盖纯音频视频，建议下一步优先补 `ASR` 适配层。

## 后续扩展建议

### 第一优先级

- 接入 `ASR` 提取原文字幕
- 增加任务取消与重试策略
- 增加批量任务队列控制

### 第二优先级

- 增加双语字幕预览页面
- 支持术语表和翻译风格模板
- 增加 `ASS` 导出

### 第三优先级

- 增加 OCR 字幕提取
- 增加剧集批量处理模板
- 增加任务执行日志页面
